# Как открыть дашборд — пошаговая инструкция (для начинающих)

Здесь всё расписано так, как будто ты никогда не открывал «терминал» и «PowerShell». Делай шаги по порядку.

---

## Что такое «терминал» и «PowerShell»

**Терминал** (или **командная строка**) — это окно в Windows, куда ты вводишь текстовые команды, а компьютер их выполняет. Никаких кнопок и меню — только текст.

**PowerShell** — это один из таких терминалов в Windows. Он уже есть на твоём компьютере, его не нужно устанавливать.

Мы будем открывать именно PowerShell и вводить в него команды по одной.

---

## Шаг 1. Открыть PowerShell

1. Нажми на клавиатуре клавишу **Windows** (значок окошка внизу слева) **или** кликни мышкой по кнопке «Пуск» в левом нижнем углу экрана.
2. Начни печатать без кавычек: **PowerShell**.
3. Когда в списке появится **Windows PowerShell** (или просто **PowerShell**), нажми на него **один раз левой кнопкой мыши**.
4. Откроется синее (или чёрное) окно с текстом вроде:
   ```
   PS C:\Users\ТвоёИмя>
   ```
   Это и есть PowerShell. Курсор мигает в конце строки — туда можно вводить команды.

**Важно:** не закрывай это окно, пока не дойдёшь до конца инструкции. Если закрыл — повтори шаг 1.

---

## Шаг 2. Перейти в папку проекта

Нужно «перейти» в папку, где лежит дашборд. Команда для этого одна.

1. В окне PowerShell введи **ровно** эту строку (можно скопировать и вставить):
   ```
   cd c:\project\anchor_checker
   ```
2. Нажми клавишу **Enter** на клавиатуре.
3. В следующей строке должно появиться что-то вроде:
   ```
   PS c:\project\anchor_checker>
   ```
   Это значит, что ты теперь «находишься» в папке проекта. Ошибок быть не должно.

**Если написало «не является внутренней или внешней командой»** — проверь, что вводил именно `cd c:\project\anchor_checker` (с пробелом после `cd`). Попробуй ещё раз.

---

## Шаг 3. Установить нужные программы (пакеты)

Команда установит программы, без которых дашборд не запустится.

1. В том же окне PowerShell введи:
   ```
   pip install -r requirements.txt
   ```
2. Нажми **Enter**.
3. Комputer начнёт что-то скачивать и писать строки в окно. Подожди, пока всё закончится. В конце снова появится строка вида:
   ```
   PS c:\project\anchor_checker>
   ```
   Тогда можно идти дальше.

---

### Шаг 3.1. Если написало, что «pip» не найден (не распознано)

Если после команды `pip install -r requirements.txt` в окне появилась **красная надпись** вроде:

- **«Имя "pip" не распознано как имя командлета...»**  
или  
- **«'pip' is not recognized...»**

это нормально. Значит, нужно вызывать установку **через Python**.

**Что сделать:**

1. В том же окне PowerShell введи **ровно** эту строку (обрати внимание: в начале **python -m pip**, с пробелами):
   ```
   python -m pip install -r requirements.txt
   ```
2. Нажми **Enter**.
3. Подожди, пока идёт установка. В конце снова должна появиться строка `PS c:\project\anchor_checker>` — тогда переходи к **Шагу 4**.

**Если и эта команда выдаёт ошибку** (например, «python не распознано») — значит, Python не установлен или не добавлен в «путь». Тогда нужно установить Python с сайта https://www.python.org/downloads/ и при установке **обязательно отметить галочку «Add Python to PATH»**, после чего снова открыть PowerShell и повторить шаги 2, 3 и 3.1.

---

## Шаг 4. Запустить дашборд

**Рекомендуемый способ** (меньше шансов на ошибку «app is not a package»):

1. В том же окне PowerShell введи:
   ```
   python run_dashboard.py
   ```
2. Нажми **Enter**.

**Альтернатива** (если уже в папке проекта):

1. В том же окне PowerShell введи:
   ```
   streamlit run app/dashboard/app.py
   ```
2. Нажми **Enter**.
3. Появится много текста. В середине или в конце ищи строки вроде:
   ```
   Local URL: http://localhost:8501
   Network URL: http://192.168...
   ```
   И ещё может быть: **You can now view your Streamlit app in your browser.**  
   Это значит, что дашборд **уже запущен**.

**Важно:** окно PowerShell **не закрывай**. Пока оно открыто — дашборд работает. Закроешь окно — страница в браузере перестанет открываться.

---

### Шаг 4.1. Если написало, что «streamlit» не найден (не распознано)

Если после команды `streamlit run app/dashboard/app.py` появилась **красная надпись** вроде:

- **«Имя "streamlit" не распознано как имя командлета...»**

значит, нужно запускать дашборд **через Python** (как с pip).

**Что сделать:** в том же окне PowerShell введи **эту** строку и нажми **Enter**:

   ```
   python -m streamlit run app/dashboard/app.py
   ```

Дождись появления строки `Local URL: http://localhost:8501`, затем открой в браузере **http://localhost:8501**.

---

**Если изначально хочешь не гадать:** можно сразу использовать команду с `python -m`:

   ```
   python -m streamlit run app/dashboard/app.py
   ```
   и нажми **Enter**.

---

## Шаг 5. Открыть дашборд в браузере

1. Открой **браузер** (Chrome, Edge, Firefox — любой).
2. В **адресной строке** (где обычно пишут google.com) введи:
   ```
   http://localhost:8501
   ```
3. Нажми **Enter**.
4. Должна открыться страница с заголовком вроде **«Дашборд: размещения ссылок по сотрудникам и проектам»** и с фильтрами (Период, чекбоксы таблиц и т.д.). Это и есть дашборд.

**Если страница не открылась:**
- Убедись, что окно PowerShell всё ещё открыто и там нет красного текста с ошибкой.
- Проверь, что в адресе именно **http://localhost:8501** (без лишних букв и пробелов).

---

## Шаг 6. Что делать на самой странице дашборда

- **Период** — выбери из списка: «Неделя», «Месяц» или «Произвольный период». От этого зависит, за какие даты показываются данные.
- Если данные подгружаются из Google Таблиц (есть файл `service_account.json` в папке проекта), через несколько секунд появятся таблицы и графики.
- Если данных нет — на странице будет написано загрузить CSV. Тогда нужно экспортировать листы из Google Таблиц в CSV (Файл → Скачать → CSV) и загрузить их кнопкой «Загрузить CSV» на этой странице.

Когда закончишь смотреть дашборд, можно просто закрыть вкладку браузера. Чтобы **полностью выключить** дашборд, в окне PowerShell нажми **Ctrl+C** (зажми Ctrl и нажми букву C), затем закрой окно PowerShell.

---

## Краткая шпаргалка (все команды подряд)

Открыл PowerShell, потом по очереди вводишь и каждый раз нажимаешь Enter:

1. `cd c:\project\anchor_checker`
2. `pip install -r requirements.txt`  
   (если не сработало: `python -m pip install -r requirements.txt`)
3. `python run_dashboard.py`  
   (или: `streamlit run app/dashboard/app.py` / `python -m streamlit run app/dashboard/app.py`)
4. В браузере открыть: **http://localhost:8501**

Окно PowerShell не закрывать, пока пользуешься дашбордом.

---

## Если что-то пошло не так

- **«pip не найден»** → используй команду с `python -m pip` (см. выше).
- **«streamlit не найден»** → используй команду с `python -m streamlit` (см. выше).
- **«python не найден»** → на компьютере не установлен Python. Нужно установить его с сайта python.org (при установке обязательно отметить «Add Python to PATH»).
- **Ошибка «No module named 'app.dashboard'; 'app' is not a package»** → запускай дашборд так: `python run_dashboard.py` (из папки `c:\project\anchor_checker`). Так Python всегда находит пакет `app`.
- **Страница в браузере не открывается** → проверь, что в PowerShell нет красного текста ошибки и что ты ввёл в адресную строку именно `http://localhost:8501`.
- **«Нет данных» на дашборде** → один раз настрой доступ по [МИНИМУМ_ДЕЙСТВИЙ.md](МИНИМУМ_ДЕЙСТВИЙ.md) (ключ + «Поделиться» в 4 таблицах). Или загрузи CSV с страницы.

Если опишешь, на каком шаге застрял и что именно написано в окне (или скинешь текст ошибки), можно будет подсказать точнее.
